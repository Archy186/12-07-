**Задание 1. Резервное копирование**
Кейс
Финансовая компания решила увеличить надёжность работы баз данных и их резервного копирования.
Необходимо описать, какие варианты резервного копирования подходят в случаях:

1.1. Необходимо восстанавливать данные в полном объёме за предыдущий день.
1.2. Необходимо восстанавливать данные за час до предполагаемой поломки.

Приведите ответ в свободной форме.

**Решение:**

*1.1. Восстановление данных в полном объёме за предыдущий день
Для этого случая идеально подходит стратегия полных резервных копий (Full Backup). Это базовый и надежный подход.
Суть метода: Создается полная копия всей базы данных. Восстановление происходит в один этап из этого полного бэкапа.
Как часто делать: Ежедневно, в часы наименьшей нагрузки. Например, каждый день в 2:00 ночи.
Пример инструментов: Для PostgreSQL это pg_dump (логическая копия) или pg_basebackup (физическая копия). Для MySQL — mysqldump или mysqlbackup.
Плюсы: Простота восстановления. Минимальный риск ошибок.*
Минусы: Требует много места для хранения (каждый бэкап — это полный объем данных). Процесс может быть долгим для больших баз данных.

*1.2. Восстановление данных за час до предполагаемой поломки
Здесь одного полного бэкапа недостаточно. Нужна стратегия "Полный бэкап + Инкрементные бэкапы" или "Полный бэкап + Бинарные логи (WAL/Binlog)".
Суть метода:
Регулярно создаются полные бэкапы (например, раз в неделю).
Между ними создаются инкрементные бэкапы, которые содержат только изменения с момента последнего бэкапа (или предыдущего инкрементального). Для MySQL это может быть команда mysqlbackup --incremental, для PostgreSQL, начиная с версии 17, — pg_basebackup --incremental.
Альтернативно, вместо инкрементных бэкапов можно непрерывно архивировать журналы транзакций (WAL в PostgreSQL, Binary Log в MySQL). Это позволяет "накатить" все изменения, произошедшие после последнего полного бэкапа.
Процесс восстановления:
Восстанавливается последний полный бэкап.
Поверх него последовательно применяются все инкрементные бэкапы (или журналы транзакций) до нужного момента времени (за час до поломки).*
